# AGENTS.md

本文件面向后续接手该项目的 Codex / coding agent。目标是让代理在读取本文件后，能够快速理解项目背景、主要约束、更新方式和迭代边界，并直接开始工作。

## 1. 项目背景

这是一个 **区域经理销售与回款周更看板** 项目。

业务场景：

- 使用者是区域经理，负责区域内医疗机构的销售与回款工作
- 日常需要关注销售表、回款表、重点医院、欠款和回款节奏
- 月底需要做销售分析、回款分析，并制定针对性的销售和回款计划
- 页面也需要兼顾向上汇报，因此既要实用，也要具备一定展示性

当前项目不是完整 BI 系统，而是一个 **离线单文件 HTML 看板**：

- 无后端
- 无登录
- 无数据库依赖
- 无外部 API
- 双击 HTML 即可打开

## 3. 当前产品定义

当前看板主视角是：

- 区域经理日常使用
- 管理层汇报可复用

固定模块：

- 顶部经营条：标题、默认窗口、刷新时间、经营摘要
- 管理总览：销售额、标本量、未回款、回款率
- 销售驾驶舱：月度经营趋势（销售额柱 + 回款率线）、销量前十医院
- 销售结构复盘：销量前十项目、三级医院最近 3 年折后销量变化
- 回款分析：账龄柱状图、重点风险医院 Top 5、某医院回款周期、医院回款预测表
- 轻量计划提示：基于当前结果自动生成 3-5 条跟进建议

明确约束：

- 医院回款预测采用 **规则预测**
- 规则预测是业务参考，不是财务承诺值
- 当前样例数据只用于验证字段和结构
- 未来真实数据接入后，区域、城市、医院、医院等级等维度必须由数据自动驱动，不能硬编码

## 4. 数据来源与更新方式

当前默认输入文件：

- `output/销售数据_模拟.xlsx`
- `output/金域应收_模拟.xlsx`

默认输出文件：

- `output/demo_dashboard_v3.html`

标准刷新命令：

```bash
python3 scripts/refresh_demo.py
```

可选自定义路径：

```bash
python3 scripts/refresh_demo.py \
  --sales /path/to/销售数据.xlsx \
  --ar /path/to/回款数据.xlsx \
  --html-out /path/to/demo_dashboard_v3.html
```

刷新脚本职责：

- 校验输入文件是否存在
- 校验 sheet 名和关键列
- 兼容部分字段别名
- 读取销售表和回款表
- 生成数据驱动的区域 / 城市 / 医院筛选
- 生成月度经营趋势、账龄结构、单医院回款周期和规则预测表
- 输出单文件离线 HTML

## 5. 已知数据口径与注意事项

现有样例数据的限制：

- 当前样例并不代表真实业务范围
- 当前样例城市不一定包含真实区域经理负责的城市
- 当前样例医院等级分布可能不完整

因此：

- 不要根据当前样例硬编码城市名、区域名或医院名
- 所有筛选项和图表维度必须从输入数据动态生成
- 迭代时优先保证“接入未来真实导出数据后能自动适配”

## 6. 修改代码时的硬约束

除非用户明确要求改架构，否则默认遵守以下原则：

- 保持 **单文件离线 HTML** 输出
- 不引入后端服务
- 不接入真实在线模型或外部 API
- 不依赖在线字体、CDN 数据请求、`fetch` 本地 JSON
- 保持 `scripts/refresh_demo.py` 为主要刷新入口
- 保留历史版本页面，不直接覆盖旧文件，优先生成新版本或明确说明替换理由

如果需要扩展数据口径：

- 优先在 `scripts/refresh_demo.py` 中补充字段映射和别名
- 优先做规则型、可解释的计算逻辑
- 避免引入难以交接的黑盒逻辑

## 7. Codex 执行建议

当用户要求你继续迭代本项目时，推荐顺序：

1. 先读 `README.md`
2. 再读 `scripts/refresh_demo.py`
3. 如涉及历史背景，再读 `docs/html看板项目汇总.md`
4. 如果是数据口径问题，再检查两份 xlsx 的表头和样例行
5. 修改后优先运行：

```bash
python3 scripts/refresh_demo.py
```

然后至少验证：

- 是否成功生成 `output/demo_dashboard_v3.html`
- HTML 是否仍为单文件离线页面
- 页面中是否没有 `fetch(` 或外链脚本依赖
- 关键模块是否仍存在：
  - 顶部经营条
  - 月度经营趋势（销售额 + 回款率）
  - 销量前十医院
  - 销量前十项目
  - 三级医院近 3 年趋势
  - 账龄柱状图
  - 重点风险医院 Top 5
  - 医院回款预测表
  - 某医院回款周期
  - 轻量计划提示

## 8. 常见迭代方向

后续较可能出现的需求：

- 接入真实系统导出的销售表 / 回款表
- 调整字段名映射
- 优化回款预测规则
- 增加区域经理更关心的经营指标
- 加强月底汇报视角
- 增加更细的医院级分析

默认处理方式：

- 先确认新字段是否能从现有导出表中直接得到
- 优先数据驱动实现
- 优先低依赖、可解释、可交接的方案
- 如果会改变看板核心定位，需要先明确是“区域经理日常优先”还是“管理层汇报优先”

## 9. 不要做的事

- 不要把样例数据里的城市、医院写死到逻辑里
- 不要把页面改成依赖网络请求才能打开
- 不要默认删除历史版本页面
- 不要把“规则预测”描述成精确预测模型
- 不要在没有必要的情况下引入新的重依赖或复杂框架

## 10. 一句话总结

这是一个 **以区域经理实际经营管理为主、兼顾管理层汇报、通过系统导出 Excel 周更的离线 HTML 看板项目**。后续迭代时，优先保证：**数据驱动、离线可用、规则清晰、便于交接**。
