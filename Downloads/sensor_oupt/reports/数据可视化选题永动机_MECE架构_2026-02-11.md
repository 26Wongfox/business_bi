# 数据可视化选题永动机：MECE架构（3.0配套）

> 目标：在“新闻价值”与“商业变现价值”双约束下，建立可持续、可复制、可扩展的选题系统。
> 适用：5人团队、多账号、分赛道运营。

---

## 1. 选题总原则（双引擎）

## 1.1 引擎A：新闻价值引擎（是否值得当下传播）
- 及时性：是否是最近7-30天内发生或升级的变化。
- 重要性：是否影响到明确人群的核心决策。
- 急迫性：用户是否“现在就要解决”。
- 重大性：是否涉及高影响（钱、机会、风险、效率）。
- 趣味性：是否具备反差、争议、惊讶点。

## 1.2 引擎B：共鸣与变现引擎（是否值得投入产能）
- 共鸣性：用户是否能说“这就是我现在的问题”。
- 信息增量：相对已有内容是否有新方法/新框架/新数据。
- 可行动性：是否可在24小时内执行第一步。
- 可传播性：是否具备“转发理由”（谈资、身份、实用）。
- 可变现性：是否能承接到模板、订阅、咨询、训练营。

说明：
- 引擎A不达标，内容很难起量。
- 引擎B不达标，内容难以长期形成商业闭环。

## 2. MECE选题架构（L0-L4）

## 2.1 L0：目标层（唯一）
- 构建“可持续增长 + 可持续变现”的数据可视化内容系统。

## 2.2 L1：一级赛道层（互斥且穷尽）

| 赛道 | 定义 | 主要价值 |
|---|---|---|
| S1 公共与社会洞察 | 宏观变化、政策、社会结构与趋势 | 时效传播、品牌权威 |
| S2 商业与增长决策 | 企业经营、营销、转化、定价、复盘 | 直接变现、B端线索 |
| S3 职场与个人决策 | 职场效率、学习、收入、职业路径 | 广谱人群、稳定流量 |
| S4 工具与方法论 | 图表工具、分析方法、可视化表达 | 模板产品、教程产品 |

MECE校验：
- 互斥：按“问题归属主体”区分（社会/企业/个人/方法）。
- 穷尽：数据可视化内容基本可落入四类之一。

## 2.3 L2：二级主题层（每个赛道下标准主题簇）

### S1 公共与社会洞察
- T1.1 人口与就业
- T1.2 城市与区域发展
- T1.3 消费与生活方式
- T1.4 教育与医疗资源

### S2 商业与增长决策
- T2.1 用户增长与留存
- T2.2 定价与利润结构
- T2.3 渠道效率与投放回报
- T2.4 经营复盘与风险预警

### S3 职场与个人决策
- T3.1 职业发展路径
- T3.2 收入结构与副业决策
- T3.3 学习效率与技能迁移
- T3.4 时间管理与决策偏差

### S4 工具与方法论
- T4.1 图表选型与叙事
- T4.2 分析框架与指标体系
- T4.3 可视化工具工作流
- T4.4 模板化资产复用

## 2.4 L3：问题场景层（触发器）

每个主题都必须绑定至少一个真实场景：
- 事件触发：政策发布、财报季、行业大会、平台规则更新。
- 决策触发：预算分配、涨价/降价、跳槽、选专业、开店。
- 风险触发：收入下滑、流量下跌、留存恶化、成本上升。
- 机会触发：新工具、新渠道、新人群、新区域。

## 2.5 L4：内容产物层（多账号适配）
- 快讯解读卡（高时效，24小时内）
- 深度图解（高收藏，方法+案例）
- 决策清单（高转化，直接行动）
- 模板演示（高商业承接，导向产品）

## 3. 选题评分模型（双引擎打分）

总分100，建议阈值：
- `>=75`：优先生产（A池）
- `60-74`：储备观察（B池）
- `<60`：暂不生产（C池）

## 3.1 新闻价值分（40分）
- 及时性 0-10
- 重要性 0-10
- 急迫性 0-8
- 重大性 0-8
- 趣味性 0-4

## 3.2 共鸣与变现分（45分）
- 共鸣性 0-10
- 信息增量 0-10
- 可行动性 0-8
- 可传播性 0-7
- 可变现性 0-10

## 3.3 可执行与风险修正（15分）
- 数据可得性 0-6
- 生产复杂度（反向）0-5
- 合规/版权风险（反向）0-4

计算方式：

```text
TopicScore = 新闻价值分(40) + 共鸣与变现分(45) + 可执行与风险修正(15)
```

## 4. 选题“永动机”运行机制（周/月）

## 4.1 周循环（固定节奏）
1. 周一上午：信号采集（新闻、平台热点、业务问题、用户反馈）。
2. 周一下午：按MECE归类到 `L1-L2`，打分入池。
3. 周二至周四：执行A池选题（同题多版本、多账号分发）。
4. 周五：复盘（传播+转化+复用性），更新评分权重。

## 4.2 月循环（结构优化）
1. 统计四赛道产能、爆款率、线索率、成交贡献。
2. 升级高表现主题簇，淘汰连续低表现方向。
3. 更新提示词库、案例库、模板库。
4. 输出下月“主打赛道 + 实验赛道”比例（建议 70%/30%）。

---

## 5. 与知识库联动（防止选题枯竭）

## 5.1 五库到选题池映射
- 素材库 -> 提供事实和数据证据。
- 灵感库 -> 提供角度和叙事假设。
- 案例库 -> 提供可复用结构和成功模式。
- 工具库 -> 提供执行路径与制作效率。
- 提示词库 -> 提供标准化生成能力。

## 5.2 最小必填字段（每个选题都要有）
- `track_id`（赛道）
- `theme_id`（二级主题）
- `scenario_trigger`（触发场景）
- `target_persona`（目标人群）
- `new_info_delta`（信息增量说明）
- `monetization_path`（变现路径）
- `content_format_plan`（产物组合）
- `topic_score`（总分）

## 7. 多层级选题池结构（执行视角）

- P0 战略池（季度）：定义赛道和主题簇，不直接生产。
- P1 月度池：每赛道5-10个候选方向，供排期。
- P2 周执行池：按分数筛出本周可做选题。
- P3 当日发布池：已完成审校、可发布内容。

规则：
- P2仅允许来自P1，禁止临时拍脑袋跳级。
- P3必须通过 `qa_review` 与 `publish_review` 两道人工审批。

---

## 8. 验收标准（判断是否成为“永动机”）

1. 连续8周不断档，每周稳定输出。
2. 四赛道中至少3个赛道持续贡献有效结果。
3. 每月有可复用模板沉淀到知识库。
4. 选题通过率（P2 -> P3）逐月提升。
5. 商业承接内容占比稳定在合理区间（建议20%-40%）。
