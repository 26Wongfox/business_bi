# 方法透明机制总则 v01

## 1. 原则
1. 透明优先：可解释性优先于表达炫技。
2. 可审计：任何关键数据都能追溯来源与处理链路。
3. 可复核：团队内他人应能复现主要结论。
4. AI可用但可控：AI参与必须留下人工复核痕迹。

## 2. 必须公开的最小信息
- 来源
- 时间范围
- 口径
- 处理步骤
- 局限性
- 更新日期

## 3. 角色责任
- Author：填写方法说明初稿
- Data Editor：校验来源与口径
- QA Reviewer：执行闸门检查
- Owner：最终签核

## 4. 透明度KPI（月度）
1. 方法说明覆盖率（目标 >= 90%）
2. 来源可追溯率（目标 = 100%）
3. 发布后勘误率（越低越好）
4. 勘误响应时长（目标 P0 < 6h）
