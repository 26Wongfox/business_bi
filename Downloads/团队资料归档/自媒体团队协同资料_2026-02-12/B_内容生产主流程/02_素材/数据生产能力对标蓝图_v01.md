# 数据生产能力对标蓝图 v01

## 1. 目标
对标 NYT/FT 等一线媒体的数据生产能力，建立“快响应 + 高可信 + 可复用”的数据生产系统，并系统使用 AI 工具提升效率。

## 2. 能力对标框架（L1-L5）

### L1 数据发现（Discovery）
- 一线能力：持续雷达监测官方数据、监管披露、行业数据库、社媒信号。
- 当前要求：建立主题化数据源池和变更提醒。
- AI增强：LLM 自动摘要数据更新公告，生成“是否值得立项”的判断草案。

### L2 数据采集（Acquisition）
- 一线能力：API优先，抓取可审计，采集动作可重放。
- 当前要求：统一采集登记、失败重试、采集日志。
- AI增强：AI 辅助写采集脚本、解析半结构化表格和 PDF。

### L3 数据处理（Processing）
- 一线能力：标准化清洗、口径对齐、版本化管理。
- 当前要求：字段字典、主键策略、缺失值策略、口径冲突记录。
- AI增强：AI 生成清洗规则初稿、异常解释建议、字段映射建议。

### L4 数据验证（Validation）
- 一线能力：多源交叉验证 + 自动化质量测试。
- 当前要求：完整性、唯一性、时效性、一致性、异常波动 5 类校验。
- AI增强：AI 先做风险扫描，再由人工审核通过。

### L5 数据发布（Delivery）
- 一线能力：面向编辑和可视化团队的标准化数据包。
- 当前要求：每次发布包含方法说明、版本号、来源引用、变更记录。
- AI增强：AI 自动生成方法说明草案和发布摘要。

## 3. 30天内的P0能力门槛
1. 数据源台账 >= 30 条且分级（A/B/C）完整。
2. 80% 的数据稿使用标准数据包（raw/clean/meta）。
3. 发布前质量闸门覆盖率 100%。
4. 每周至少 1 次数据生产复盘。

## 4. 数据生产组织（建议）
1. Data Editor：口径、来源、质量最终责任。
2. Data Producer：采集与清洗执行。
3. AI Operator：提示词与自动化维护。
4. QA Reviewer：质量测试和发布前签核。

## 5. 技术与工具栈（建议）
1. 采集：API + Playwright + 手工补采。
2. 处理：SQL + Python + CSV 模板。
3. 质检：规则校验脚本 + AI 风险扫描。
4. 协同：标准模板 + 版本日志 + 发布闸门。
5. AI层：用于提速、补全、初筛，不替代最终签核。
