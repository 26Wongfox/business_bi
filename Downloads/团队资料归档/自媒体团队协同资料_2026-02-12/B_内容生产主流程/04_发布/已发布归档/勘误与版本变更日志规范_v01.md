# 勘误与版本变更日志规范 v01

## 1. 目标
确保发布后任何修订都可追溯、可解释、可审计。

## 2. 触发勘误的情形
1. 数据来源错误或链接失效。
2. 指标口径错误或计算错误。
3. 图表与正文不一致。
4. 方法说明遗漏关键限制。

## 3. 勘误等级
- P0（严重）：影响核心结论，需立即下线修复。
- P1（中等）：影响部分结论，24小时内修复。
- P2（轻微）：不影响结论，72小时内修复。

## 4. 变更日志字段
- content_id
- account
- version_before
- version_after
- change_level（P0/P1/P2）
- change_type（数据/文案/图表/方法说明）
- change_summary
- impact_scope
- changed_by
- reviewed_by
- change_time
- public_note

## 5. 对外披露规范
1. P0/P1 必须在正文末尾追加“更新说明”。
2. 更新说明应包含：更新时间、修改项、是否影响结论。
3. 禁止静默改动核心数字。

## 6. 模板（Markdown）
```md
### 更新说明（YYYY-MM-DD HH:mm）
- 修改项：
- 原因：
- 是否影响结论：是/否
- 版本：vX -> vY
```
