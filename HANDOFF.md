# HANDOFF.md

## 项目概览

这是一个 **区域经理销售与回款周更看板** 项目，当前交付形式是 **离线单文件 HTML 页面**。

适用场景：

- 区域经理日常查看销售、回款、欠款和重点医院
- 月底做销售分析和回款分析
- 生成针对性的销售与回款计划
- 向管理层做经营汇报

当前不是完整 BI 系统，而是一个便于更新和分享的 Demo / 轻量经营驾驶舱：

- 无后端
- 无数据库服务
- 无登录
- 无外部 API 依赖
- 双击 HTML 可直接打开

## 当前看板包含的内容

当前 V3 页面固定包含：

1. 管理总览
   - 顶部经营条
   - 折后销售额
   - 标本量
   - 未回款金额
   - 回款率

2. 销售分析
   - 月度经营趋势（销售额柱 + 回款率线）
   - 销量前十医院
   - 销量前十项目
   - 三级医院最近 3 年折后销量变化

3. 回款分析
   - 账龄柱状图
   - 重点风险医院 Top 5
   - 某医院回款周期分析
   - 医院回款预测表（默认前 8 行，可展开）

4. 轻量计划提示
   - 自动生成 3-5 条本周 / 本月跟进建议

## 业务口径

这个项目当前主视角是：

- 区域经理日常使用
- 管理层汇报可复用

回款预测说明：

- 当前采用 **规则预测**
- 这是业务跟进参考，不是财务承诺值
- 不要把页面里的预测结果表述成精确模型输出

## 如何更新看板

### 默认方式

1. 用最新系统导出文件覆盖：

- `output/销售数据_模拟.xlsx`
- `output/金域应收_模拟.xlsx`

2. 在仓库根目录运行：

```bash
python3 scripts/refresh_demo.py
```

3. 刷新成功后打开：

- `output/demo_dashboard_v3.html`

### 自定义路径方式

如果新数据不放在默认位置，可以用命令行参数：

```bash
python3 scripts/refresh_demo.py \
  --sales /path/to/销售数据.xlsx \
  --ar /path/to/回款数据.xlsx \
  --html-out /path/to/demo_dashboard_v3.html
```

## 刷新脚本做了什么

`scripts/refresh_demo.py` 会：

- 识别项目根目录
- 校验输入文件是否存在
- 校验 sheet 名和关键列
- 兼容部分字段别名
- 读取销售与回款数据
- 动态生成区域 / 城市 / 医院筛选
- 生成月度经营趋势、账龄结构、单医院回款周期和规则预测表
- 输出离线单文件 HTML

默认输出：

- `output/demo_dashboard_v3.html`

## 常见问题

### 1. 提示缺少输入文件

说明：

- 默认数据文件不在 `output/` 下
- 或命令行传入路径错误

### 2. 提示缺少工作表

当前脚本默认优先读取：

- `销售数据_模拟.xlsx -> 销售数据源`
- `金域应收_模拟.xlsx -> 金域应收`

如果文件里只有一个 sheet，脚本会自动回退到唯一工作表。

### 3. 提示缺少关键列

说明：

- 新的系统导出模板字段变了
- 当前脚本虽有部分字段别名兼容，但不一定覆盖全部情况

处理方式：

- 检查新导出表表头
- 在 `scripts/refresh_demo.py` 里补充字段别名映射
