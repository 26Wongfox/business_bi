# 区域经理销售与回款周更看板

本项目当前以 **区域经理日常使用 + 管理层汇报可复用** 的离线 HTML 看板为核心交付，重点解决每周更新销售和回款数据、每天跟踪重点医院、月底做销售与回款分析的问题。

## 当前主要产物

- 最新周更看板：`output/demo_dashboard_v3.html`
- 上一版演示页面：`output/demo_dashboard_v2.html`
- 历史演示页面：`output/demo_dashboard.html`
- 数据刷新脚本：`scripts/refresh_demo.py`
- 历史数据脚本：`scripts/build_demo_json.py`
- 项目总结文档：`docs/html看板项目汇总.md`

## 看板内容

当前新版看板固定包含：

- 顶部经营条：标题、默认窗口、刷新时间、经营摘要
- 管理总览：销售额、标本量、未回款、回款率
- 销售驾驶舱：月度经营趋势（销售额柱 + 回款率线）、销量前十医院
- 销售结构复盘：销量前十项目、三级医院最近 3 年折后销量变化
- 回款分析：账龄柱状图、重点风险医院 Top 5、某医院回款周期、医院回款预测表
- 轻量计划提示：基于当前结果自动生成 3-5 条跟进建议

说明：

- 医院回款预测采用 **规则预测**，是业务跟进参考，不是财务承诺值。
- 当前仓库样例数据只用于验证字段和结构，未来真实系统导出数据接入后，城市、医院、等级等维度都按数据自动驱动。

## 推荐使用方式

新版看板是单文件离线 HTML，直接双击即可打开：

- `output/demo_dashboard_v3.html`

如果只是快速预览，也可以在仓库根目录启动静态服务：

```bash
python3 -m http.server 8000
```

然后浏览器访问：

- `http://localhost:8000/output/demo_dashboard_v3.html`

## 如何用新数据更新看板

后续数据仍采用“系统导出 -> 覆盖固定文件名 -> 一键刷新”的方式。

### 1. 覆盖固定输入文件

把最新系统导出数据覆盖到：

- `output/销售数据_模拟.xlsx`
- `output/金域应收_模拟.xlsx`

### 2. 执行刷新命令

在任意工作目录都可以运行，脚本会基于自身所在仓库自动解析默认路径：

```bash
python3 /Users/huangzhikuan/Documents/business_intellgent/my-landing-page-v3/scripts/refresh_demo.py
```

脚本会自动完成这些动作：

- 校验输入文件是否存在
- 校验 sheet 名和关键列
- 对部分表头别名做兼容
- 读取销售和回款数据
- 动态生成区域、城市、医院、医院等级等维度
- 生成月度经营趋势、账龄结构、单医院回款周期和回款预测表
- 将图表库和数据直接内嵌进 `output/demo_dashboard_v3.html`
- 在终端打印刷新结果、默认时间窗口和关键指标摘要

### 2.1 可选：覆盖默认路径

如果接手的人把导出文件放在别的目录，可以直接传参：

```bash
python3 scripts/refresh_demo.py \
  --sales /path/to/销售数据.xlsx \
  --ar /path/to/回款数据.xlsx \
  --html-out /path/to/demo_dashboard_v3.html
```

### 3. 打开输出文件

刷新成功后，直接打开：

- `output/demo_dashboard_v3.html`

## 常见错误

- `缺少输入文件`
  说明导出文件没有放到默认位置，或传入路径不正确。

- `缺少工作表`
  当前脚本默认优先读取：
  `销售数据_模拟.xlsx -> 销售数据源`
  `金域应收_模拟.xlsx -> 金域应收`
  如果文件内只有一个 sheet，脚本会自动回退到唯一工作表。

- `缺少关键列`
  说明新的系统导出模板字段发生了变化。脚本已内置一部分别名兼容；如果仍报错，需要补充字段映射。
