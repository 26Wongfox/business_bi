# 金域医学 HTML 看板项目汇总

## 1. 项目目标与需求范围

本阶段目标是快速交付一个可演示、可分享、可在手机浏览器打开的 BI 看板 Demo，围绕以下核心业务问题：

- 管理层总览：销售规模、标本量、未回款、短期回款预测
- 销售分析：Top10 医院、Top10 项目、趋势变化
- 回款与风险：账龄结构、DSO 风险医院
- 商机线索：结合政策/资讯提取线索并可视化

本阶段明确不包含：

- Java 运行与服务端部署说明
- 历史数据库部署方案说明

## 3. 开发过程与思路梳理

### 3.1 数据到页面的路径

采用“先汇总、后渲染”的思路，确保页面逻辑简单、渲染稳定：

1. 将业务指标整理成统一 JSON 结构（KPI、趋势、排行、风险、线索）
2. 前端页面只负责读取 JSON 并渲染
3. 在移动端场景下，提供“离线单文件版”避免跨文件加载失败

### 3.2 页面结构设计

看板采用分层结构，优先保证管理层阅读效率：

- 第一层：4 个 KPI 卡片（总览）
- 第二层：销售趋势 + 账龄结构（核心态势）
- 第三层：Top 医院/Top 项目（结构贡献）
- 第四层：风险与线索（行动优先级）

### 3.3 视觉风格选择（医疗风）

设计原则：清洁、可信、低认知负担。

- 主色系：蓝绿医疗配色
- 背景：轻纹理与柔和渐变，降低视觉噪音
- 文本层级：标题/说明/数据三层分离
- 图表风格：统一网格线、统一字体、统一动画节奏

### 3.4 交互与动态能力

为提升演示价值，加入轻量可交互机制：

- 时间范围筛选（起止月份）
- 趋势指标切换（折后销售额 / 标本量）
- 商机线索关键词过滤
- KPI 数字递增动画
- 图表入场动画

### 3.5 移动端与分享问题处理

问题：直接用 `file://` 打开时，浏览器常拦截 JSON 请求或外部脚本。

处理：生成单文件离线版 `demo_dashboard_embedded.html`：

- 内嵌图表库脚本
- 内嵌完整业务数据
- 无需本地服务、无需网络即可在手机打开
